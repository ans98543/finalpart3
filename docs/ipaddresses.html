IP Addresses, Packets and Routing

IP addresses allow networks to communicate with one another. In 1970, Vint Cerf and Bob Khan were the creators of this method. Thanks to them, billions of networks are interconnected. The internet is set in a standard protocol. Due to this new technology we only need to know which protocol to use to connect. Each device has an IP address and this consists of a code of numbers. IP is the internet protocol that sends info back to one another. The structure of addresses evolved over time. The creators of the internet also created a system to transfer data from one device to another by packets.

IP packets can send info, but their info has to split it up and once all packets reach their destination the outcome is formed. In these packets, it contains 1s and 0s. Routers are the main reason packets are able to take different destinations. The philosophy to have more than direction a packet can save packets from getting jammed up. Similar to a traffic jam during rush hour. The TCP role is to identify each packet and to make sure everything arrived. If something is missing authorization will not be approved.
